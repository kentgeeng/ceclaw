## 架構說明（重要）

### 雙層 AI 架構
```
OpenClaw :18789  → agent 模式 → 管理/工具用途（IT/Admin）
Hermes   :3000   → proxy :8000 → RAG → 員工日常對話
```
**不是 bug，是設計正確。** Hermes UI 才是員工使用介面。

### RAG 查詢順序
```
① L3 Qdrant（ceclaw_*）+ L2 tw_knowledge → 並行，threshold 0.7
② L1 law_advisor_api → _LAW_KEYWORDS 條件執行
③ inject SOUL.md → LLM
```

### 三層知識庫
```
L1: tw_laws（221,599條，18類，GB10 Qdrant）
L2: tw_knowledge（51,970筆，12類，GB10 Qdrant）
L3: ceclaw_*（GB10 Qdrant，ceclaw_ 前綴隔離多租戶）
    - ceclaw_personal_{user_id}
    - ceclaw_dept_{dept_name}
    - ceclaw_company_{company_id}
```

### SearXNG Adapter
```
Hermes web_search → Firecrawl API（/v1/search + /v2/search）
    → searxng_adapter.py :2337
    → SearXNG :8888
```

```bash

cd ~/ceclaw && .venv/bin/python3 -c "from router import proxy; print('_KS_AVAILABLE:', proxy._KS_AVAILABLE)"

# 2. 直接測 query_all_layers
cd ~/ceclaw && .venv/bin/python3 - << 'EOF'
import asyncio
from router.knowledge_service_v2 import query_all_layers
async def test():
    hits = await query_all_layers("測試查詢")
    print(f"hits: {len(hits)}")
    for h in hits:
        print(f"  score={h['similarity']} content={h['content'][:60]}")
asyncio.run(test())
EOF

# 3. 看 log
```
