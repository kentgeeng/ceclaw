## ⚠️ 未完成核心問題（必修）

### 🔴 問題1：No models available
**根本原因：** `/api/models`（workspace server）有 `isAuthenticated` 檢查，回 401 → models 空
**關鍵檔案：**
- `~/hermes-workspace/src/routes/api/models.ts` 第107行
- `~/hermes-workspace/src/server/auth-middleware.ts`

**Debug：**
```bash
# 確認 /api/models 回什麼
curl -sv http://172.25.0.12:3000/api/models 2>&1 | head -30

# 看 auth 邏輯
cat ~/hermes-workspace/src/server/auth-middleware.ts

# 找 bypass 機制
grep -n "isAuthenticated\|bypass\|dev\|local\|DEV\|DISABLE" ~/hermes-workspace/src/server/auth-middleware.ts
grep -n "AUTH\|DISABLE" ~/hermes-workspace/.env 2>/dev/null

# :8642/v1/models 是正常的
curl -s http://localhost:8642/v1/models
```

**修法（從簡到繁）：**
- A：`auth-middleware.ts` 加 localhost bypass
- B：找 .env 的 AUTH 開關
- C：`models.ts` 拿掉 `isAuthenticated` 檢查（workspace 是 local only）

---

### 🔴 問題2：chat stream「reading 'slice'」
**根本原因：** SSE 某個 event 的 data 是 undefined，`use-streaming-message.ts` 解析時 `.slice()` 爆炸

**關鍵檔案：**
- `~/hermes-agent-fork/gateway/platforms/api_server.py` 約1088行 `_handle_session_chat_stream`
- `~/hermes-workspace/src/server/hermes-api.ts` 第300行 `streamChat()`
- `~/hermes-workspace/src/screens/chat/hooks/use-streaming-message.ts` 第396行

**workspace 接受的 data 格式：**
```javascript
// 第396行：chunk.delta ?? chunk.text ?? chunk.content ?? chunk.chunk ?? ''
```
必須包含 `delta` 或 `text` 或 `content` 或 `chunk`

**正確 SSE 格式：**
```
event: message
data: {"delta": "文字片段"}

event: message
data: {"delta": "更多文字"}

data: [DONE]

```

**Debug：**
```bash
# 手動測 stream
curl -s -N -X POST http://localhost:8642/api/sessions/test/chat/stream \
  -H "Content-Type: application/json" \
  -d '{"message": "你好"}' --max-time 120

# 看 workspace SSE 解析邏輯
sed -n '670,690p' ~/hermes-workspace/src/screens/chat/hooks/use-streaming-message.ts
sed -n '390,400p' ~/hermes-workspace/src/screens/chat/hooks/use-streaming-message.ts
```
