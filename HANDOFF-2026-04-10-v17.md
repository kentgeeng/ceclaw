## 重要技術細節

### vault 記憶層（本次新增）
```
路徑：~/ceclaw/vault/（symlink → ~/.ceclaw/vault/）
四個 md 檔：
  working-context.md  ← Hermes 現在在做什麼
  project-state.md    ← 所有專案進度
  decisions-log.md    ← 重要決策記錄
  daily/YYYY-MM-DD.md ← 每日工作日誌

讀寫規則（在 ~/ceclaw/config/SOUL.md）：
  - 每次對話開始：讀 working-context.md + project-state.md
  - 每 3-5 次 tool call：更新 working-context.md
  - 任務完成：append daily log，更新 project-state.md
  - 重要決策：append decisions-log.md
```

### Hermes P3 hook 位置
```python
# 現在：~/hermes-agent-fork/webapi/routes/chat.py
# 第 338-357 行
# P3: auto-submit to shared_bridge if task completed with tools
# 條件：result.get("api_calls", 0) > 1

# B70 後升級到 v0.8.0 時，新位置：
# gateway/platforms/api_server.py
# _handle_chat_completions 函數內
# result, usage = await _compute_completion() 之後
```

### upstream 策略
| 元件 | 策略 | 說明 |
|------|------|------|
| OpenClaw | 永久 fork（選項 A）| 等重大安全漏洞才升級 |
| Hermes | 選項 B→C | B70 升級時把 P3 hook 改成 builtin_hooks/ |
| proxy.py | 完全自主 | 不依賴任何 upstream |

### CDC OTA 升級流程
```
upstream 新版本
  ↓
CDC 測試環境（pop-os + GB10）72 小時燒機
驗證：P3 hook / RAG 三層 / shared_bridge / vault
  ↓ 通過
客戶 OTA：openshell sandbox update --name ceclaw-companyA
```
