# CECLAW 記憶庫架構設計
**狀態：Draft — 量產前實作**
**更新：2026-04-06**

## 五層架構

個人記憶（Hermes，絕對私有）
  本人 + CEO授權者可讀，IT/主管/其他人完全看不到
        ↓ 員工主動貢獻
        shared_bridge 提交
              ↓ 部門內審核
        部門知識（dept_eng / dept_hr）
        部門資料庫（IT直接匯入）
        同部門全員可讀
              ↓ 管理員採納
        公司知識（員工貢獻）
        公司共同資料庫（IT直接匯入）
        全員可讀

## 五層存取控制

個人記憶    本人+CEO授權者讀  Hermes自動寫
部門知識    同部門全員讀      部門內審核
公司知識    全員讀            IT/管理員採納
部門資料庫  同部門全員讀      IT匯入
公司資料庫  全員讀            IT匯入

## 個人記憶存取原則

本人：讀寫
CEO或CEO授權者：唯讀＋審計log（法律要求）
IT管理員：無內容存取權，只管基礎設施
主管：完全看不到
其他人：完全看不到

## CEO授權鏈（依客戶需求選擇）

A. 靜態授權名單（ceclaw-policy.yaml）一般企業
B. 即時批准（MFA）高安全需求
C. 限時授權（法務調查期間）法律/合規調查
A+C 名單+有效期限，最常見組合
onboard時由客戶IT+法務確認選擇

## 檔案格式

全層統一 Parquet + DuckDB VSS（語意搜尋）
橋接記錄 Parquet，完整歷史可追溯
